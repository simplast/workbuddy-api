## 完整问题描述（存档）

### H1. `src/routes/openai.js:233` — 流式 watchdog 触发后重复 `res.end()`

watchdog 触发时会调用 `res.end()` 并 `clearInterval(watchdog)`，但外层 `try` 块的 `while` 循环仍在 `await reader.read()`。reader 取消后循环退出，进入 `finally` (清 watchdog，OK)，再回到 line 233 调用 `writeDataEvent('[DONE]')` 在已经 `res.end()` 的连接上写数据，触发 `ERR_STREAM_WRITE_AFTER_END`。

### H2. 客户端中途断开未传播到 upstream

两个流式端点都没有监听 `req.on('close')` / `res.on('close')`。

### H3. `src/routes/anthropic.js:325-339` — pseudo-XML 流式响应下重复发块

流过程中已经把 `delta.content` 累积的伪 XML 通过 `text_delta` 发给客户端，stream 结束后再发 tool_use 块，重复且 stop_reason 被改写。
